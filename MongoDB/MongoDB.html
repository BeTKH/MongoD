3. FINDING DOCUMENTS 

[.find(), .findOne()]

dbname>db.collname.findOne() // returns single document – the 1st

dbname>db.collname.find()    // prints the 1st 20 docs

dbname>itr                   // prints the nxt 20 docs

3.1 FILTER

dbname>db.collname.find({})  // {} is a filter

example:

cinimax>movies.find({title:”The Matrix”})

3.2 FILTER + FILEDS 

dbname>db.collname.find({},{})  // {} filter, {} fild selector is flag

example: // return only title and rating fields

cinimax>movies.find({title:”The Matrix”}, {title:1, rating:1})

dbname>db.collname.find({},{}).count()  // {} filter, {} fild selector

3.4 FILTER + FILEDS + LIMIT

dbname>db.collname.find({},{}).limit(3)  // get only 3 documents

3.5 FILTER + FILEDS + SORT + LIMIT

dbname>db.collname.find({},{}).sort({title:1})  //  sort is  1 = asc

dbname>db.collname.find({},{}).sort({title:-1})  // sort is -1 = dsc

dbname>db.collname.find({},{}).sort({title:-1}).limit(3)

4. INSERTING NESTED DOCUMETS

.insertOne(), .insertMany()

> db.collname.insertOne({key:[{key:val},..., {key:val}[a]]})

> db.collname.insertMany([{key:[{key:val},..., {key:val}[b]]},{k:v}])

5. FINDING USING OPERATORS

operators begin with $ sign

5.1 numerical operators: [$gt, $lt, $gte, $lte]

> db.collname.find({key:{$gt:num}})

// find a movie w rating > 8

eg: cinimax.movies.find({rating:{$gt:8}})

5.2 logical operators: [OR, AND, NOT]

> db.collname.find({$or:[{k:v}, {k:v}]}) … one of the listing mus be T

// find a movie w rating 7 or title “Matrix”

Eg. cinimax.movies.find({$or:[rating:7, title:”Matrix”]})

5.3 numerical + logical operators:

> db.collname.find({$or:[{k:{$gt:num}}, {k:{$lt:num}}])

// find a movie w rating > 7 or <3

Eg. cinimax.movies.find({$or:[rating:{$gt:7}, rating:{$lt:3}]})

5.4 $in & $nin operators

$in - a certain field is within the list.

$nin - a certain field is NOT in the list.

E.g.

//find a movie with a rating 7 or 8 or 9

cinimax.movies.find({$or:[{rating:7},{rating:8},{rating:9}]})

//a rating $in [7,8,9]

cinimax.movies.find({rating:{$in:[7,8,9]}})

//a rating $nin [7,8,9]

cinimax.movies.find({rating:{$nin:[7,8,9]}})

6. QUERING DOCS with fields having ARRAY as value

>Lets say we have a collection of movies with 2 docs like:

        [        

        {title:”The Matrix”, gener:[”Sci-fi”, ”action”]},

{title:”Terminator”, gener:[”action”]},

{title:”Terminator”, gener:[”action”]}

]

6.1 getting a doc if a value is present in the array

>specify any value present in the array

>returns all files with the values present in them

eg.

// get movies with action gener

cinimax.movies.find({gener:”action”})

// returns both The Matrix and Terminator

6.2 getting a doc only with a specified value in the array

>Specify filter value as an arry

>returns a doc with exact match in the array

eg.

// get movies gener exactly action, nothing else

cinimax.movies.find({gener:[”action”]})

// returns only Terminator

6.2 using operator $all

>gets all documents that contain specified array elements and more if any.

eg.

// get movies with specified gener plus any other movie

cinimax.movies.find({gener:{$all:[”action”, ”action”]}})

7. QUERING NESTED DOCS

>Nested docs have field/property names nested.

>To access sub property names . must be used.

db.collname.find({”outer_prop.inner_prop”:”valuOf_nestedProp”})

eg. // get movies with specific reviewer name

cinimax.movies.find({”reviews.name”:”max”})

8. DELETING DOCS from a COLLECTION

[.deleteOne(), .deleteMany()]

db.collname.deleteOne({Key:val})

db.collname.deleteMany({key:val})

> delete by _id is the best practice when deleting one.

examples:

// delete a movie with id_

db.collname.deleteOne({_id:ObjectId(“622shxvhjvxsa41”)})

// delete all movies with rating = 9

cinimax.movies.deleteMany({rating:9})
